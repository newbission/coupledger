# coupledger

함께(또는 혼자) 쓰는 **생활비 카드 정산 · 가계부** 웹앱.
카드 이용내역 엑셀을 올리면 거래를 공용/개인으로 분류하고, 한 사람이 결제한 생활비를 깔끔하게 정산해 줍니다.

### 🔗 바로가기 → **https://newbission.github.io/coupledger/**

> **couple + ledger** — 가운데 `le`를 함께 씁니다. 커플을 위해 만들었지만, 가족도·룸메도·혼자여도 누구나.

---

## 주요 기능

- 📤 **엑셀 업로드** — 삼성카드 '카드이용내역' `.xlsx` 자동 파싱 (취소 netting, 전체취소 제외, 할부 표시). 어댑터 구조라 다른 카드/은행도 추가 가능.
- 👥 **N명 멤버** — 공용 지출은 가중치로 분배, 결제자 1명이 전액 결제 후 나머지가 정산. 혼자면 가계부 모드.
- 🏷️ **카테고리** — 가맹점 빈도학습으로 자동제안(AI 없이 쓸수록 똑똑) + 수정/추가 + 한 거래를 여러 카테고리로 **분할**.
- ✍️ **거래 직접 추가** — 현금 등 엑셀에 없는 항목 수동 입력, 수동 제외/복구.
- 🔃 **정렬** — 일자·가맹점·카테고리·금액 헤더 클릭 정렬.
- 💾 **내보내기/기록** — CSV 내보내기, JSON 백업(브라우저 간 이동), 월별 정산 **기록 저장·불러오기**.
- 🎨 **테마** — 따뜻함 · 미니멀 · 슬레이트 3종 전환.

## 데이터 & 프라이버시

- **백엔드 없음.** 모든 데이터(설정·내역·기록)는 **사용자 브라우저의 localStorage**에만 저장됩니다. 서버로 전송되는 것은 아무것도 없습니다.
- 업로드한 카드 내역 파일도 **브라우저 안에서만** 파싱됩니다.
- 기기/브라우저를 옮길 땐 **설정 → 백업 내보내기/가져오기**(JSON)로 이동하세요.

## 로컬 실행

```bash
npm install
npm run dev      # 개발 서버 (http://localhost:5173)
npm run build    # 프로덕션 빌드 → dist/
npm run preview  # 빌드 결과 미리보기
```

## 기술 스택

- **Vite + TypeScript** (프레임워크 없음), 100% 정적 사이트
- 엑셀 파싱: **SheetJS**
- 테마: CSS 디자인 토큰(`data-theme`)
- 배포: **GitHub Actions → GitHub Pages**

## 사용 흐름

1. 첫 실행 → 온보딩(카드 선택 · 멤버 · 테마 설정)
2. 메인 → 카드 내역 `.xlsx` 업로드
3. 각 거래를 공용/멤버로 분류 + 카테고리 확인
4. 정산 결과 확인 → CSV 내보내기 / 이번 달 기록 저장
